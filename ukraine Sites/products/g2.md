## G2.com Listing & Pagination Mechanism — Full Breakdown

---

### 1. Rendering Model: Server-Side HTML + PJAX

G2 category pages are **server-side rendered (SSR)**. The initial page load at `/categories/crm` delivers a complete HTML document — no lazy-loaded API calls are needed for the first set of listings. There is **no separate JSON/GraphQL API** for listings.

When the user changes a filter or pagination, G2 uses **jQuery PJAX** (pushState + AJAX). PJAX fetches the new page via XHR, swaps only the `#ajax-container` div (not a full page reload), and updates the browser URL via `history.pushState`. This means:

- **First load** → normal GET, full HTML response
- **Subsequent filter/page changes** → XHR request to the same URL, server returns full HTML, but only `#ajax-container` content is swapped in the DOM

---

### 2. Request Structure

#### Base URL pattern
```
https://www.g2.com/categories/{slug}
```

#### PJAX XHR Headers sent on every filter/pagination change
```
X-PJAX: true
X-PJAX-Container: #ajax-container
X-Requested-With: XMLHttpRequest
X-CSRF-Token: <token>
```

The server detects the `X-PJAX` header and returns **only the inner HTML** for `#ajax-container` (not a full page). If you call the URL **without** that header (plain GET), you get a full HTML page — useful for scraping.

---

### 3. URL / Query String Parameters

All filters and pagination are passed as **GET query string parameters**:

| Parameter | Values | Description |
|---|---|---|
| `order` | `g2_score`, `popular`, `top_shelf` | Sort order |
| `page` | `1`, `2`, … `63` | Page number |
| `filters[star_rating][]` | `5`, `4`, `3` | Star rating filter (multi-select) |
| `filters[languages_supported][]` | `19`, `78`, `25`, `17`… (integer IDs) | Language filter |
| `filters[pricing_options][]` | `1` (Pricing Available), `2` (Deals Available) | Pricing filter |
| `filters[verified_features][]` | integer IDs | Feature filter |
| `filters[Standalone or All in One][]` | `36` (Standalone), `35` (All-in-One) | Deployment type |
| `utf8` | `✓` | Rails UTF-8 hidden field (can be omitted) |

#### Segment filter — separate URL path (not query param!)
Market segment is **not** a query param. It changes the URL path:

```
/categories/crm                  → All Segments (942 results)
/categories/crm/small-business   → Small Business (173 results)
/categories/crm/mid-market       → Mid Market (111 results)
/categories/crm/enterprise       → Enterprise (52 results)
```

---

### 4. Pagination Details

- **15 items per page** (confirmed: 942 total ÷ 63 pages ≈ 14.95)
- Pagination controls are `<ul class="pagination">` with standard numbered `<a>` links
- "Prev/Next" links are also present
- The paginator shows: `‹ Prev | 1 | 2 | 3 | 4 | 5 | … | 62 | 63 | Next ›`

---

### 5. How to Scrape Listings Programmatically

Since the content is SSR and the full HTML is available on plain GET (no browser needed for most cases), you can directly fetch pages:

```
# Page 1, default sort
GET https://www.g2.com/categories/crm?order=g2_score&page=1

# Page 2
GET https://www.g2.com/categories/crm?order=g2_score&page=2

# With filters: 5-star only, Small Business segment, page 3
GET https://www.g2.com/categories/crm/small-business?order=g2_score&page=3&filters[star_rating][]=5

# Multiple filters (4+5 star, pricing available)
GET https://www.g2.com/categories/crm?order=g2_score&page=1&filters[star_rating][]=5&filters[star_rating][]=4&filters[pricing_options][]=1
```

**To get only the listing fragment** (like PJAX does, faster/lighter response), add these headers:
```
X-PJAX: true
X-PJAX-Container: #ajax-container
X-Requested-With: XMLHttpRequest
```

This makes the server return only the swapped `#ajax-container` HTML, skipping nav, footer, etc. — a smaller payload. Note you'll also need a valid `X-CSRF-Token` for this to work server-side (grab it from any page's `<meta name="csrf-token">` tag first).

---

### 6. Summary Flow

```
User visits /categories/crm
    → Full SSR HTML delivered (15 listings in #ajax-container)
    → PJAX initializes on .pjax-form + .pagination links

User clicks filter checkbox / page number
    → Form serialized as GET params
    → XHR GET to same URL with X-PJAX headers
    → Server returns inner HTML only
    → jQuery PJAX swaps #ajax-container
    → URL updated via history.pushState (bookmarkable/crawlable URL)
```
