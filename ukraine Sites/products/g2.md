## G2.com Listing & Pagination Mechanism — Full Breakdown

---

### 1. Rendering Model: Server-Side HTML + PJAX

G2 category pages are **server-side rendered (SSR)**. The initial page load at `/categories/crm` delivers a complete HTML document — no lazy-loaded API calls are needed for the first set of listings. There is **no separate JSON/GraphQL API** for listings.

When the user changes a filter or pagination, G2 uses **jQuery PJAX** (pushState + AJAX). PJAX fetches the new page via XHR, swaps only the `#ajax-container` div (not a full page reload), and updates the browser URL via `history.pushState`. This means:

- **First load** → normal GET, full HTML response
- **Subsequent filter/page changes** → XHR request to the same URL, server returns full HTML, but only `#ajax-container` content is swapped in the DOM

---

### 2. Request Structure

#### Base URL pattern
```
https://www.g2.com/categories/{slug}
```

#### PJAX XHR Headers sent on every filter/pagination change
```
X-PJAX: true
X-PJAX-Container: #ajax-container
X-Requested-With: XMLHttpRequest
X-CSRF-Token: <token>
```

The server detects the `X-PJAX` header and returns **only the inner HTML** for `#ajax-container` (not a full page). If you call the URL **without** that header (plain GET), you get a full HTML page — useful for scraping.

---

### 3. URL / Query String Parameters

All filters and pagination are passed as **GET query string parameters**:

| Parameter | Values | Description |
|---|---|---|
| `order` | `g2_score`, `popular`, `top_shelf` | Sort order |
| `page` | `1`, `2`, … `63` | Page number |
| `filters[star_rating][]` | `5`, `4`, `3` | Star rating filter (multi-select) |
| `filters[languages_supported][]` | `19`, `78`, `25`, `17`… (integer IDs) | Language filter |
| `filters[pricing_options][]` | `1` (Pricing Available), `2` (Deals Available) | Pricing filter |
| `filters[verified_features][]` | integer IDs | Feature filter |
| `filters[Standalone or All in One][]` | `36` (Standalone), `35` (All-in-One) | Deployment type |
| `utf8` | `✓` | Rails UTF-8 hidden field (can be omitted) |

#### Segment filter — separate URL path (not query param!)
Market segment is **not** a query param. It changes the URL path:

```
/categories/crm                  → All Segments (942 results)
/categories/crm/small-business   → Small Business (173 results)
/categories/crm/mid-market       → Mid Market (111 results)
/categories/crm/enterprise       → Enterprise (52 results)
```

---

### 4. Pagination Details

- **15 items per page** (confirmed: 942 total ÷ 63 pages ≈ 14.95)
- Pagination controls are `<ul class="pagination">` with standard numbered `<a>` links
- "Prev/Next" links are also present
- The paginator shows: `‹ Prev | 1 | 2 | 3 | 4 | 5 | … | 62 | 63 | Next ›`

---

### 5. How to Scrape Listings Programmatically

Since the content is SSR and the full HTML is available on plain GET (no browser needed for most cases), you can directly fetch pages:

```
# Page 1, default sort
GET https://www.g2.com/categories/crm?order=g2_score&page=1

# Page 2
GET https://www.g2.com/categories/crm?order=g2_score&page=2

# With filters: 5-star only, Small Business segment, page 3
GET https://www.g2.com/categories/crm/small-business?order=g2_score&page=3&filters[star_rating][]=5

# Multiple filters (4+5 star, pricing available)
GET https://www.g2.com/categories/crm?order=g2_score&page=1&filters[star_rating][]=5&filters[star_rating][]=4&filters[pricing_options][]=1
```

**To get only the listing fragment** (like PJAX does, faster/lighter response), add these headers:
```
X-PJAX: true
X-PJAX-Container: #ajax-container
X-Requested-With: XMLHttpRequest
```

This makes the server return only the swapped `#ajax-container` HTML, skipping nav, footer, etc. — a smaller payload. Note you'll also need a valid `X-CSRF-Token` for this to work server-side (grab it from any page's `<meta name="csrf-token">` tag first).

---

### 6. Summary Flow

```
User visits /categories/crm
    → Full SSR HTML delivered (15 listings in #ajax-container)
    → PJAX initializes on .pjax-form + .pagination links

User clicks filter checkbox / page number
    → Form serialized as GET params
    → XHR GET to same URL with X-PJAX headers
    → Server returns inner HTML only
    → jQuery PJAX swaps #ajax-container
    → URL updated via history.pushState (bookmarkable/crawlable URL)
```

## Блок: Страница отзывов продукта — Reviews & Pagination

---

### 1. URL продуктовой страницы отзывов

```
https://www.g2.com/products/{product-slug}/reviews
```

Пример:
```
https://www.g2.com/products/salesforce-salesforce-sales-cloud/reviews
```

Это **полноценная SSR-страница** — доступна plain GET без PJAX-заголовков. Внутри есть якорь `#reviews`, к которому ведут все ссылки пагинации и фильтров.

---

### 2. Параметры URL — фильтры и пагинация отзывов

Все фильтры и пагинация передаются через GET-параметры. Якорь `#reviews` добавляется к URL на клиенте — для скрапинга он не нужен.

| Параметр | Значения | Описание |
|---|---|---|
| `page` | 1, 2, … | Номер страницы отзывов |
| `order` | `g2_default`, `most_recent`, `most_helpful`, `highest_rated`, `lowest_rated` | Сортировка |
| `filters[nps_score]` | `5`, `4`, `3`, `2`, `1` | Фильтр по звёздам (radio, single-select) |
| `filters[comment_answer_values]` | строка | Полнотекстовый поиск по тексту отзывов |
| `filters[company_segment][]` | `179` (Small Business), `180` (Mid-Market), `181` (Enterprise) | Размер компании (multi-select) |
| `filters[role][]` | `1` (User), `2` (Administrator), `3` (Executive Sponsor), `4` (Internal Consultant), `6` (Consultant), `7` (Agency), `8` (Industry Analyst / Tech Writer) | Роль рецензента (multi-select) |
| `filters[category_ids][]` | integer IDs | Категория продукта |
| `filters[industry][]` | integer IDs | Отрасль компании-рецензента |
| `filters[region][]` | `North America`, `Asia`, `Europe`, `Latin America`, `ANZ`, `Middle East`, `Africa` | Регион |

Примеры:
```
# Страница 2, сортировка по дате
GET /products/{slug}/reviews?order=most_recent&page=2

# Только 5-звёздочные отзывы
GET /products/{slug}/reviews?filters[nps_score]=5

# Mid-Market + Administrator, страница 3
GET /products/{slug}/reviews?filters[company_segment][]=180&filters[role][]=2&page=3

# Поиск по ключевому слову
GET /products/{slug}/reviews?filters[comment_answer_values]=automation
```

> **Важно:** `filters[nps_score]` — **radio**, не checkbox. Передаётся одно значение, без `[]`.

---

### 3. Rendering-модель фильтров

Фильтры работают через тот же **PJAX-механизм**, что и на страницах категорий. Форма фильтров имеет `id="review-filters"` и класс `pjax-form`. Все фильтрующие элементы (включая те, что физически лежат вне тега `<form>`) привязаны к форме через атрибут `form="review-filters"` и `data-submit-on-change-form="#review-filters"`, т.е. при изменении любого контрола автоматически отправляется форма.

Сортировка (`<select name="order">`) обёрнута в виджет **Choices.js** (скрытый `<select>` + кастомный дропдаун), но name и form-binding сохраняются.

### 6. Рейтинг в SVG: логика заполнения

В новом дизайне G2 звёзды рендерятся как **5 SVG-элементов** с `<path>`. Цвет заливки:

| Состояние | fill |
|---|---|
| Полная звезда (filled) | `#FF492C` (фирменный оранжево-красный G2) |
| Пустая звезда (empty) | `#DFDFE2` (светло-серый) |

Числовой рейтинг дублируется в тексте `<label>` рядом со звёздами в формате `N/5` или `N.5/5`.

### 7. Пагинация отзывов

```html
<ul class="pagination" role="navigation" aria-label="Pagination">

  <!-- Prev (disabled на стр. 1) -->
  <li class="pagination__component pagination__component--disabled">
    <span class="pagination__only-nessy">‹ Prev</span>
  </li>

  <!-- Текущая страница (без ссылки) -->
  <li class="pagination__component pagination__page-number pagination__page-number--current">
    1
  </li>

  <!-- Числовые страницы -->
  <li class="pagination__component pagination__page-number show-for-large">
    <a class="pagination__page-number-link js-log-click pjax"
       href="/products/{slug}/reviews?page=2#reviews">2</a>
  </li>
  <li class="pagination__component pagination__page-number show-for-large">
    <a href="/products/{slug}/reviews?page=3#reviews">3</a>
  </li>
  <!-- ... -->

  <!-- Gap (многоточие) -->
  <li class="pagination__component pagination__component-gap">…</li>

  <!-- Последние страницы (без show-for-large) -->
  <li class="pagination__component pagination__page-number show-for-large">
    <a href="/products/{slug}/reviews?page=9#reviews">9</a>
  </li>
  <li class="pagination__component pagination__page-number show-for-large">
    <a href="/products/{slug}/reviews?page=10#reviews">10</a>
  </li>

  <!-- Next -->
  <li class="pagination__component">
    <a class="pagination__named-link js-log-click pjax"
       href="/products/{slug}/reviews?page=2#reviews">
      <img>
      <span>Next ›</span>
    </a>
  </li>

</ul>
```

**Детали пагинации отзывов:**

- **10 отзывов на странице** (в отличие от 15 на страницах категорий)
- Параметр пагинации: `?page=N` (без `order` по умолчанию)
- Якорь `#reviews` добавляется к href всех ссылок пагинации
- Структура классов пагинатора идентична категорийным страницам
- Фильтры сохраняются в ссылках пагинации (сервер добавляет их к href)

**Расчёт страниц для Salesforce Sales Cloud:**
```
25,449 отзывов ÷ 10 на страницу = 2,545 страниц
```

### 8. Ключевые CSS-селекторы для парсинга отзывов

| Что вытащить | CSS-селектор |
|---|---|
| Список отзывов (контейнер) | `#reviews article` |
| Профиль рецензента (ссылка) | `article a[href*="/users/"]` |
| Имя рецензента | `article a[href*="/users/"] + div div.elv-tracking-normal:first-child` |
| Должность | `article .elv-text-xs:nth-of-type(1)` |
| Размер компании | `article .elv-text-xs:nth-of-type(2)` |
| Дата отзыва | `article label` → текст с паттерном `M/D/YYYY` |
| Рейтинг (числовой) | `article label` → текст с паттерном `N/5` или `N.5/5` |
| Рейтинг (SVG-звёзды) | `article svg path` → `fill="#FF492C"` = заполнена, `fill="#DFDFE2"` = пуста |
| Заголовок отзыва | `article .elv-text-xl.elv-font-semibold` |
| Текст отзыва | `article .spht` → ищи родительский `<div>` |
| Бейдж верификации | `article button label.elv-font-semibold` |
| Общий счётчик отзывов | `#reviews h2` → текст вида `Reviews (N)` |
| Текущая страница | `ul.pagination .pagination__page-number--current` |
| Ссылка "Next" | `ul.pagination a.pagination__named-link` |
| Числовые ссылки пагинации | `ul.pagination a.pagination__page-number-link` |
| Ссылка "Video Reviews" | `a[href*="video-reviews"]` |
