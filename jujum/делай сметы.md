Чтобы программно настроить синтезатор (написать скрипт, создать контроллер или сохранить пресет), простого имени и значения (0.0–1.0) недостаточно. Вам нужна мета-информация, которая позволит понять, **как именно** интерпретировать эти числа.

Вот полный список информации, которую необходимо извлекать из параметров VST-синтезатора для грамотного управления:

### 1. Базовые атрибуты (Обязательно)
Без этого вы не сможете даже отобразить ручку управления.

*   **Parameter ID (Уникальный идентификатор):**
    *   *Зачем:* Индексы параметров (0, 1, 2...) могут меняться при обновлении плагина или смене пресета. ID (часто `int` или `string`) остается постоянным.
    *   *Пример:* `param_id = 1024` (для Cutoff).
*   **Parameter Name (Имя):**
    *   *Зачем:* Для отображения в интерфейсе.
    *   *Пример:* `"Cutoff Frequency"`.
*   **Normalized Value (Нормализованное значение):**
    *   *Зачем:* Внутренний формат VST (от `0.0` до `1.0`). Используется для передачи данных в плагин.
    *   *Пример:* `0.5`.
*   **Display Value (Текстовое представление):**
    *   *Зачем:* Чтобы пользователь видел понятные единицы измерения.
    *   *Пример:* `"1200 Hz"`, `"-3.5 dB"`, `"On"`.

### 2. Атрибуты для маппинга (Критично для синтеза)
Синтезаторы используют нелинейные шкалы. Если вы сделаете линейный слайдер для частоты фильтра, управлять им будет невозможно (все изменения будут в начале пути).

*   **Min/Max Range (Диапазон):**
    *   *Зачем:* Чтобы знать физические границы.
    *   *Пример:* Cutoff от `20 Hz` до `20000 Hz`.
*   **Scaling / Curve (Шкала / Кривая):**
    *   *Зачем:* Понимать, линейная зависимость или логарифмическая.
    *   *Важно:* Частоты (Freq), Время (Time) и Громкость (Gain) почти всегда **логарифмические**.
    *   *Пример:* Значение `0.5` может означать не среднюю частоту (10000 Гц), а среднюю по логарифму (примерно 630 Гц).
*   **Step Count (Дискретность):**
    *   *Зачем:* Понимать, параметр непрерывный (ручка) или шаговый (переключатель).
    *   *Пример:*
        *   `0` шагов = Плавная ручка (Filter Cutoff).
        *   `3` шага = Переключатель (Saw, Square, Sine).
        *   `1` шаг = Вкл/Выкл (Bypass).

### 3. Структурная информация (Для организации UI)
Чтобы не вываливать 200 параметров в одну кучу, нужно знать, к какой части синтезатора они относятся.

*   **Category / Group (Группа):**
    *   *Зачем:* Разделить параметры на осцилляторы, фильтры, огибающие, эффекты.
    *   *Пример:* `"Oscillator 1"`, `"Filter"`, `"Envelope 2"`.
*   **Parameter Type (Тип):**
    *   *Зачем:* Понимать, как рисовать контрол.
    *   *Пример:* `Knob` (ручка), `Switch` (кнопка), `Menu` (выпадающий список).

### Как получить эту информацию в C++ (JUCE)

В JUCE класс `AudioProcessorParameter` предоставляет методы для получения большинства из этих данных.

```cpp
void analyzeSynthParameter(juce::AudioProcessorParameter* param)
{
    // 1. Идентификация
    int index = param->getIndex();
    juce::String name = param->getName(64);
    // В VST3 лучше использовать getParamID(), если доступно через VST3Specifics
    
    // 2. Текущее состояние
    float normValue = param->getValue(); // 0.0 - 1.0
    juce::String display = param->getCurrentValueAsText(); // "1.2 kHz"
    
    // 3. Диапазон и шкала (Самое важное для синтеза!)
    // JUCE хранит информацию о том, как конвертировать 0-1 в реальные единицы
    auto range = param->getNormalisableRange(); 
    
    float minReal = range.start;   // Например, 20.0
    float maxReal = range.end;     // Например, 20000.0
    float skew = range.skew;       // Коэффициент искривления (1.0 = линейно, !=1.0 = логарифм)
    
    // 4. Дискретность
    int steps = param->getNumSteps(); 
    bool isDiscrete = (steps > 0);
    
    // Вывод отладки
    DBG("Param: " << name);
    DBG("  Range: " << minReal << " to " << maxReal);
    DBG("  Skew: " << skew << " (1=Linear, <1=Log)");
    DBG("  Steps: " << (isDiscrete ? juce::String(steps) : "Continuous"));
    DBG("  Current: " << display);
}
```

### 4. Специфические параметры для настройки звука
Если ваша цель — **автоматическая настройка звука** (например, подгонка под частоту ноты), вам нужно искать параметры по ключевым словам. Вот список того, что обычно нужно контролировать:

| Группа | Ключевые слова для поиска (Name) | Тип данных | Примечание |
| :--- | :--- | :--- | :--- |
| **Oscillator** | `Wave`, `Shape`, `Type`, `Saw`, `Square` | Discrete (Menu) | Выбор формы волны |
| **Pitch** | `Tune`, `Pitch`, `Detune`, `Coarse`, `Fine` | Continuous | Частота в полутонах или центах |
| **Filter** | `Cutoff`, `Freq`, `Frequency` | Continuous (Log) | Частота среза (обязательно лог-шкала) |
| **Filter** | `Res`, `Resonance`, `Q` | Continuous | Резонанс |
| **Filter** | `Type`, `Mode`, `LP`, `HP`, `BP` | Discrete (Menu) | Тип фильтра |
| **Envelope** | `A`, `D`, `S`, `R`, `Attack`, `Decay`... | Continuous (Log) | Время в секундах/мс (лог-шкала) |
| **LFO** | `Rate`, `Speed`, `Sync` | Continuous/Discrete | Скорость модуляции |
| **Master** | `Volume`, `Gain`, `Level` | Continuous (Log) | Громкость |
| **Unison** | `Voices`, `Detune`, `Spread` | Mixed | Количество голосов и расстройка |

### 5. Важные нюансы при настройке

1.  **Сглаживание (Smoothing):**
    *   При программном изменении параметров (особенно `Cutoff` или `Pitch`) во время воспроизведения звука, резкое изменение вызовет "щелчок" (click/pop).
    *   *Решение:* Изменяйте значение постепенно (интерполяция) в течение 10–50 мс, либо используйте встроенное сглаживание плагина, если оно доступно.

2.  **Зависимость от сэмпла (Sample Rate):**
    *   Параметры времени (Attack, Release, LFO Rate) могут зависеть от частоты дискретизации (44.1k vs 48k).
    *   *Решение:* Всегда ориентируйтесь на **Display Value** (секунды/Гц), а не на сырое значение 0.0–1.0, так как плагин сам пересчитает нормализованное значение под текущий Sample Rate.

3.  **Пресеты (Presets):**
    *   Если вы хотите сохранить настройку, не пытайтесь сохранять значения каждого параметра по отдельности.
    *   *Решение:* Используйте `AudioProcessor::getStateInformation()` (в JUCE), чтобы получить полный бинарный дамп состояния плагина (XML или бинарные данные). Это надежнее, чем запоминать 150 ручек.

### Итог: Алгоритм действий
1.  **Сканируйте** все параметры плагина.
2.  **Фильтруйте** их по именам (ищите "Cutoff", "Attack", "Wave" и т.д.).
3.  **Проверяйте `getNormalisableRange()`**, чтобы понять, как мапить вашу мышку/слайдер на значение 0.0–1.0 (используйте `convertTo0to1()` и `convertFrom0to1()` из JUCE).
4.  **Проверяйте `getNumSteps()`**, чтобы решить, рисовать ручку или выпадающий список.
5.  **Используйте Parameter ID** для сохранения привязок, а не индекс.

Если вы сделаете это, ваш инструмент сможет адекватно управлять любым синтезатором, понимая разницу между линейной громкостью и логарифмической частотой.
